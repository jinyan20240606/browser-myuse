# Sync 模块

Sync 模块负责浏览器配置文件（Profile）的同步和认证管理，支持将本地浏览器配置同步到云端，实现跨设备的一致性体验。

## 目录结构

```
sync/
├── service.py          # 同步服务实现
├── auth.py             # 认证管理
├── __init__.py         # 模块导出
└── README.md           # 本文件
```

## 核心文件详解

### [`service.py`](service.py)
**核心类：** `SyncService`
- **作用：** 管理浏览器配置文件的同步
- **功能：**
  - 上传本地配置到云端
  - 从云端下载配置
  - 合并配置变更
  - 冲突解决

### [`auth.py`](auth.py)
**核心类：** `AuthManager`
- **作用：** 管理认证和授权
- **功能：**
  - 用户认证
  - Token 管理
  - 权限验证
  - 会话管理

## 核心功能

### 配置同步
- **Cookie 同步：** 保持登录状态
- **书签同步：** 同步浏览器书签
- **历史记录：** 同步浏览历史
- **扩展设置：** 同步扩展配置

### 认证管理
- **OAuth 集成：** 支持多种 OAuth 提供商
- **API 密钥：** 安全管理 API 密钥
- **会话持久化：** 保持登录会话

## 使用场景

### 云端部署
将本地浏览器配置同步到云端，在远程浏览器中使用相同的登录状态和设置。

```python
from browser_use import Browser

# 使用云端配置
browser = Browser(
    cloud_profile_id='your-profile-id'
)
```

### 跨设备同步
在不同设备间保持一致的浏览器状态。

### 团队协作
共享浏览器配置，便于团队协作。

## 与其他模块的关系

- **与 Browser 模块：** 同步浏览器配置文件
- **与 Cloud 模块：** 通过云端服务进行同步
- **与 Sandbox 模块：** 在沙箱环境中使用同步的配置

## 调用链路

```
本地配置 → SyncService → 云端存储 → 远程浏览器
```

## 安全考虑

- 所有传输数据都经过加密
- 敏感信息（如密码）不同步
- 支持选择性同步
- 提供配置备份和恢复功能

## 配置选项

- `sync_enabled`: 是否启用同步
- `sync_interval`: 同步间隔（秒）
- `sync_cookies`: 是否同步 Cookie
- `sync_bookmarks`: 是否同步书签
